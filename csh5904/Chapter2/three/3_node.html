1. ROS 2 노드(Node)의 개념

  • 정의: ROS 2에서 노드는 특정 단일 목적의 작업(센서 데이터 수집, 모터 제어, 알고리즘 연산 등)을 수행하는 최소 단위의 실행 프로그램(프로세스)입니다.
  • 구조적 특징: 하나의 큰 프로그램이 로봇 전체를 제어하는 것이 아니라, 기능별로 쪼개진 수많은 노드들이 독립적으로 실행되면서 서로 유기적으로 데이터를 주고받는 분산형 아키텍처를 취합니다.

2. 노드(Node) 단위 개발의 필요성 및 장점

  • 모듈화와 재사용성: 센서나 모터 구동 노드를 독립적으로 만들어 두면 하드웨어가 바뀌어도 해당 노드만 교체하거나 재사용할 수 있습니다.
  • 시스템 결함 격리: 하나의 노드에서 에러가 발생해 다운되더라도 다른 노드들은 독립된 프로세스이므로 영향을 받지 않고 정상 구동이 가능합니다.
  • 다국어 개발: 각 노드가 독립된 프로세스이기 때문에 기능에 따라 C++, Python 등 서로 다른 프로그래밍 언어로 작성하여 연동할 수 있습니다.

3. 노드 간의 통신 메커니즘

  1. 토픽 (Topic): 1:N 방식의 연속적인 데이터 스트림 통신 (발행/구독 구조).
  2. 서비스 (Service): 1:1 방식의 동기식 요청/응답 통신.
  3. 액션 (Action): 목표 지정, 중간 피드백, 최종 결과 반환 구조의 비동기식/장시간 작업용 통신.
  4. 파라미터 (Parameter): 노드 내부의 설정값을 외부에서 실시간으로 변경할 수 있도록 하는 노드별 개별 변수.

4. 터틀심(Turtlesim) 데모의 노드 간 토픽 통신 예시

  • 속도 명령 전달: 터틀 텔레옵(turtle_teleop) 노드는 키보드 입력을 받아 거북이의 속도 제어 명령을 /cmd_vel 토픽에 담아 터틀(turtlesim) 노드에게 발행(Publish)합니다.
  • 상태 및 피드백 전달: 터틀(turtlesim) 노드는 거북이의 실시간 거동 결과에 따른 피드백 토픽과 현재 상태(위치, 자세 등) 토픽을 터틀 텔레옵(turtle_teleop) 노드에게 발행하여 전달합니다.
