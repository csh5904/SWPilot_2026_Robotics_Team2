# 📄 1_linux.md

## 1. 셸 환경이 실행되는 터미널 실행 방법

* **개념:** 터미널(Terminal)은 사용자가 글자를 입력해 리눅스 시스템과 소통하는 인터페이스(창)이며, 셸(Shell)은 사용자가 입력한 명령어를 해석하여 운영체제 커널(Kernel)에 전달하고 실행 결과를 다시 사용자에게 보여주는 프로그램입니다.
* **실행 방법:** 우분투 리눅스 환경에서 단축키 `Ctrl` + `Alt` + `T`를 누르면 즉시 터미널 창이 기본 셸과 함께 실행됩니다.

## 2. 홈 디렉토리의 개념

* **개념:** 리눅스 시스템에 로그인한 사용자(User)별로 주어지는 고유의 개인 저장 및 작업 공간입니다.
* **특징:** 경로 상에서 물결표(`~`) 기호로 축약하여 표현하며, 절대 경로로는 `/home/사용자계정이름`에 위치합니다. 시스템 핵심 파일들이 위치한 영역을 건드리지 않고 사용자가 자유롭게 파일과 폴더를 제어할 수 있는 안전한 격리 공간입니다.

## 3. 파일과 디렉토리를 관리하는데 필요한 필수 명령어

리눅스 시스템 데이터 제어를 위한 핵심 명령어 테이블입니다.

| 명령어 | 기능 및 설명 |
| --- | --- |
| `pwd` | **Print Working Directory:** 현재 세션이 위치해 있는 디렉토리의 전체 절대 경로를 화면에 출력합니다. |
| `ls` | **List:** 현재 디렉토리 안에 있는 파일과 폴더 목록을 보여줍니다. `ls -al`을 사용하면 숨겨진 파일(`.`)과 상세 권한, 소유자까지 롱 포맷으로 한눈에 볼 수 있습니다. |
| `cd [경로]` | **Change Directory:** 지정한 경로의 폴더로 위치를 이동합니다. 인자 없이 `cd`나 `cd ~`만 입력하면 사용자의 홈 디렉토리로 즉시 복귀합니다. `cd ..`은 상위 폴더로 이동합니다. |
| `mkdir [폴더명]` | **Make Directory:** 현재 위치에 새로운 폴더(디렉토리)를 생성합니다. |
| `rm [파일명]` | **Remove:** 지정한 파일을 완전히 삭제합니다. 폴더를 내부 내용물까지 통째로 강제 삭제하려면 재귀 옵션을 더해 `rm -rf [폴더명]` 구조를 사용합니다. |
| `cp [원본] [대상]` | **Copy:** 파일이나 디렉토리를 복사합니다. 폴더를 복사할 때는 하위 구조 보존을 위해 `cp -r` 옵션을 사용합니다. |
| `mv [원본] [대상]` | **Move / Rename:** 파일이나 디렉토리의 위치를 이동시키거나 이름을 변경할 때 사용합니다. |

## 4. 파일의 내용을 확인하고 편집하는 방법

* **파일 내용 확인 (`cat`):** 터미널 창에서 `cat 파일명` 명령어를 입력하면 파일의 내부 텍스트 내용을 수정 없이 가볍고 빠르게 터미널 화면에 출력해 볼 수 있습니다.
* **파일 텍스트 편집 (`nano`):** GUI(화면 그래픽 인터페이스)가 없는 환경에서도 직관적으로 코드를 수정할 수 있는 터미널 내장 텍스트 편집기입니다.
* `nano 파일명`으로 편집 창에 진입합니다.
* 수정 완료 후 `Ctrl` + `O` (저장) ➡️ `Enter` (파일명 확인) ➡️ `Ctrl` + `X` (종료) 순서로 안전하게 빠져나옵니다.



## 5. 파일의 권한에 대한 개념과 파일의 권한을 바꾸는 방법

* **권한 개념:** 리눅스는 보안을 위해 모든 파일과 폴더에 **읽기(r, 4), 쓰기(w, 2), 실행(x, 1)** 권한을 부여하며, 이 권한을 독립적으로 `[소유자(User) / 그룹(Group) / 기타 사용자(Others)]` 세 단계로 나누어 통제합니다.
* **권한 변경 (`chmod`):** 파일의 접근 설정을 바꿀 때는 `chmod` 명령어를 사용합니다.
* 특정 파일에 실행 권한만 추가할 때: `chmod +x 파일명`
* 숫자 모드로 소유자에게 모든 권한(`rwx` = 7), 그룹과 기타 사용자에게 읽기/실행 권한(`r-x` = 5)을 줄 때: `chmod 755 파일명`



## 6. 실행파일, 셸 스크립트의 개념과 source 명령어 사용법

* **실행파일:** 시스템이 구조를 인지하여 직접 메모리에 올려 작업을 수행할 수 있도록 실행 속성(`x`)이 인가된 파일입니다.
* **셸 스크립트:** 리눅스 터미널에서 연속으로 실행할 일련의 명령어들을 하나의 텍스트 파일(확장자 `.sh`)에 순서대로 적어두고 순차 실행시키는 스크립트 자동화 파일입니다.
* **source 명령어:** 셸 스크립트를 실행할 때, 스크립트 내부에서 설정되거나 바뀐 환경 변수값이나 함수 정의를 **현재 열려 있는 터미널 창(현재 로그인 세션 셸)에 즉시 동기화하여 적용**시키고자 할 때 사용합니다. (예: `source ~/.bashrc`)

## 7. 사용자 권한, 슈퍼유저의 개념과 sudo 명령어

* **사용자 권한 vs 슈퍼유저 (Root):** 일반 사용자 계정은 보안상 운영체제 핵심 영역을 훼손할 수 없도록 자신의 홈 디렉토리 외에 시스템 설정을 바꿀 수 없습니다. 반면 슈퍼유저(Root)는 시스템의 모든 파일, 프로세스, 하드웨어를 제어할 수 있는 무한한 최고 관리자 권한입니다.
* **sudo (SuperUser Do) 명령어:** 일반 사용자 계정에서 패키지 설치나 시스템 파일 수정 등 관리자 권한이 일시적으로 필요할 때 명령어 맨 앞에 붙이는 명령어입니다. 실행 시 본인 계정의 비밀번호 확인을 요구합니다.

## 8. 우분투 리눅스의 패키지 관리 방법과 apt 명령어

* **패키지 관리 개념:** 스마트폰의 앱스토어처럼, 프로그램의 설치·업데이트·의존성 제어·삭제를 인터넷 원격 저장소와 연동하여 자동으로 관리해 주는 편리한 시스템입니다.
* **apt 명령어 사용 절차:**
1. `sudo apt update`: 원격 저장소의 최신 프로그램 리스트 및 버전 정보를 로컬 저장소 카탈로그에 동기화합니다. (새로운 패키지 설치 전 필수 진행)
2. `sudo apt install [패키지명]`: 동기화된 리스트를 바탕으로 지정한 소프트웨어 패키지를 시스템에 자동으로 다운로드하여 빌드 및 설치를 수행합니다.

## 9. 리눅스 환경 심층 문제 이해 및 운영체제 비교 분석

### Q1. 우분투의 `/etc` 디렉토리에 포함된 파일들을 확인하는 것은 자유로우나, 지우거나 수정하는 데 제한이 있는 이유는 무엇인가요?

**A1.** `/etc` 디렉토리는 리눅스 시스템 전반의 핵심 하드웨어 네트워크 설정, 사용자 계정 정보, 패키지 실행 환경 설정 파일들이 모여 있는 **시스템의 제어실**입니다. 일반 사용자가 이 영역의 파일을 실수나 악의적인 의도로 수정 및 삭제할 경우 시스템 전반이 마비될 수 있습니다.
따라서 리눅스는 다중 사용자 보안 모델에 따라 일반 사용자에게는 오직 읽기(Read) 권한만 허용하고 쓰기 및 삭제 권한을 제한합니다. 이 구역에서 파일을 생성, 편집, 삭제하려면 반드시 `sudo` 명령어를 결합하여 최고 관리자(Root) 권한을 일시적으로 위임받아 실행해야만 합니다.

### Q2. `bash`란 무엇인가요?

**A2.** bash(Bourne Again Shell)는 리눅스 환경에서 사용되는 여러 명령어 해석기(Shell) 중 하나로, 유닉스의 표준 셸 기능에 명령어 자동 완성(Tab), 히스토리 저장, 강력한 스크립팅 기능을 추가해 개선한 셸입니다. 리눅스 커널 커뮤니티의 표준 셸이기도 하며, **우분투 리눅스의 경우 사용자가 터미널을 열었을 때 구동되는 '기본 셸(Default Shell)'로 bash를 채택**하여 처리하고 있습니다.

### Q3. 리눅스 운영체제의 프로그램 실행 과정은 윈도우(Windows)와 어떻게 다른가요?

**A3.** 확장자와 경로 탐색 방식에서 두 운영체제는 명확한 차이를 보입니다.

* **윈도우:** `.exe`, `.bat` 등 파일 이름의 확장자를 보고 실행 파일인지 식별합니다. 또한 파일이 위치한 현재 폴더가 기본적으로 경로 탐색 범위에 포함되므로, 현재 디렉토리 안에 있는 프로그램은 파일 이름만 타이핑해도 즉시 실행됩니다.
* **리눅스:** 파일 확장자와 상관없이 오직 파일 메타데이터 내부의 **실행 권한(`x`)** 유무로만 실행 여부를 결정합니다. 보안 문제로 인해 리눅스는 현재 디렉토리(`.`)를 환경 변수 경로 탐색 범위(`$PATH`)에서 기본 제외하므로, 현재 위치에 있는 실행 파일이나 셸 스크립트를 작동할 때도 반드시 `./파일명`과 같이 **현재 위치 경로 지시자 및 파일 이름 전체를 명시**해야만 구동됩니다.

### Q4. 파이썬 프로그램(`1_hello.py`)에서 파일을 홈 디렉토리에 생성할 때와 최상위 `/test` 디렉토리를 생성하여 파일 작업을 할 때 어떤 점이 다르며 어떻게 해결해야 하나요?

**A4.** 접근하려는 대상 경로에 소유 권한이 있는지의 차이입니다.

* **홈 디렉토리(`~`) 생성:** 일반 사용자가 독점적인 소유권을 가진 안전 구역이므로 파이썬 스크립트가 별도의 권한 요청 없이도 자유롭게 폴더를 신설하고 파일을 작성할 수 있습니다.
* **루트 아래 `/test` 디렉토리 생성:** 루트 디렉토리(`/`) 하위 구역은 슈퍼유저(Root)만 수정할 수 있는 시스템 공용 관리 구역입니다. 일반 사용자 권한으로 단순히 `python3 1_hello.py`를 실행하여 이곳에 직접 디렉토리나 파일을 만들려고 하면 운영체제 단에서 권한을 차단해 `Permission Denied(권한 거부)` 에러가 발생합니다.
* **해결 방법:** 이를 해결하기 위해서는 스크립트를 구동하기 전 터미널에서 `sudo mkdir /test`로 관리자 권한 폴더를 선제 생성한 후, 모든 권한을 일반 유저에게 여는 `sudo chmod 777 /test` 작업을 수행해 두거나, 파이썬 프로그램 실행 자체를 최고 관리자 권한으로 승격해 `sudo python3 1_hello.py`로 명령을 수행해야 권한 충돌 없이 정상 동작합니다.
