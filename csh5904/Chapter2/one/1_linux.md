
# 📄 1_linux.md

## 1. 셸 환경이 실행되는 터미널 실행 방법

* **개념:** 터미널은 사용자가 글자를 입력해 리눅스 시스템과 소통하는 창(인터페이스)이며, 셸(Shell)은 사용자가 입력한 명령어를 해석하여 운영체제 커널에 전달하고 실행하는 프로그램입니다.
* **실행 방법:** 우분투 리눅스 환경에서 단축키 `Ctrl` + `Alt` + `T`를 누르면 즉시 터미널 창이 실행됩니다.

## 2. 홈 디렉토리의 개념

* **개념:** 리눅스 시스템에 로그인한 사용자(User)별로 주어지는 고유의 개인 저장 및 작업 공간입니다.
* **특징:** 경로 상에서 물결표(`~`) 기호로 축약하여 표현하며, 절대 경로로는 `/home/사용자계정이름`에 위치합니다. 시스템 핵심 영역을 건드리지 않고 사용자가 자유롭게 파일과 폴더를 제어할 수 있는 안전한 공간입니다.

## 3. 파일과 디렉토리를 관리하는데 필요한 명령어

* `pwd` (Print Working Directory): 현재 내가 위치해 있는 디렉토리의 전체 절대 경로를 화면에 출력합니다.
* `ls` (List): 현재 디렉토리 안에 있는 파일과 폴더 목록을 보여줍니다. `ls -al`을 사용하면 숨겨진 파일과 상세 권한까지 한눈에 볼 수 있습니다.
* `cd [경로]` (Change Directory): 지정한 경로의 폴더로 위치를 이동합니다. 인자 없이 `cd`나 `cd ~`만 입력하면 홈 디렉토리로 즉시 복귀합니다.
* `mkdir [폴더명]` (Make Directory): 현재 위치에 새로운 폴더(디렉토리)를 생성합니다.
* `rm [파일명]` (Remove): 지정한 파일을 삭제합니다. 폴더를 내부 내용까지 통째로 삭제하려면 `rm -rf [폴더명]` 구조를 사용합니다.

## 4. 파일의 내용을 확인하고 편집하는 방법

* **파일 내용 확인 (cat):** 터미널 창에서 `cat 파일명` 명령어를 입력하면 파일의 내부 텍스트 내용을 수정 없이 가볍게 출력해 볼 수 있습니다.
* **파일 텍스트 편집 (nano):** GUI(화면 인터페이스)가 없는 터미널 환경에서도 직관적으로 코드를 수정할 수 있는 내장 텍스트 편집기입니다.
* `nano 파일명`으로 편집 창에 진입합니다.
* 수정 완료 후 `Ctrl` + `O` (저장) ➡️ `Enter` (파일명 확인) ➡️ `Ctrl` + `X` (종료) 순서로 빠져나옵니다.



## 5. 파일의 권한에 대한 개념과 파일의 권한을 바꾸는 방법

* **권한 개념:** 리눅스는 보안을 위해 모든 파일과 폴더에 **읽기(r, 4), 쓰기(w, 2), 실행(x, 1)** 권한을 부여하며, 이 권한을 `[소유자 / 그룹 / 기타 사용자]` 세 단계로 나누어 통제합니다.
* **권한 변경 (chmod):** 파일의 권한 설정을 바꿀 때는 `chmod` 명령어를 사용합니다.
* 특정 파일에 실행 권한을 추가할 때: `chmod +x 파일명`
* 숫자 모드로 소유자에게 모든 권한(7), 그룹과 기타 사용자에게 읽기/실행 권한(5)을 줄 때: `chmod 755 파일명`



## 6. 실행파일, 셸 스크립트의 개념과 source 명령어 사용법

* **실행파일:** 시스템이 인지하여 직접 메모리에 올려 작업을 수행할 수 있도록 실행(x) 권한이 인가된 파일입니다.
* **셸 스크립트:** 리눅스 터미널에서 연속으로 실행할 명령어들을 하나의 텍스트 파일(확장자 `.sh`)에 순서대로 적어두고 순차 실행시키는 스크립트 파일입니다.
* **source 명령어:** 셸 스크립트를 실행할 때, 스크립트 내부에서 설정되거나 바뀐 환경 변수값들을 **현재 열려 있는 터미널 창(세션)에 즉시 동기화하여 적용**시키고자 할 때 사용합니다.

## 7. 사용자 권한, 슈퍼유저의 개념과 sudo 명령어

* **사용자 권한 vs 슈퍼유저 (Root):** 일반 사용자 계정은 보안상 자신의 홈 디렉토리 외에 시스템 핵심 설정을 바꿀 수 없습니다. 반면 슈퍼유저(Root)는 시스템의 모든 파일과 프로세스를 제어할 수 있는 전지전능한 최고 관리자 권한입니다.
* **sudo (SuperUser Do) 명령어:** 일반 사용자 계정에서 패키지 설치나 시스템 파일 수정 등 관리자 권한이 일시적으로 필요할 때 명령어 맨 앞에 붙이는 치트키입니다. 실행 시 본인 계정의 비밀번호 확인을 요구합니다.

## 8. 우분투 리눅스의 패키지 관리 방법과 apt 명령어

* **패키지 관리 개념:** 스마트폰의 앱스토어처럼, 프로그램의 설치·업데이트·삭제를 인터넷 저장소와 연동하여 자동으로 관리해 주는 시스템입니다.
* **apt 명령어 사용 절차:**
* `sudo apt update`: 원격 저장소의 최신 프로그램 리스트 및 버전 정보를 로컬에 동기화합니다. (프로그램 설치 전 필수 진행)
* `sudo apt install [패키지명]`: 동기화된 리스트를 바탕으로 원하는 소프트웨어를 시스템에 자동으로 다운로드하여 설치합니다.
