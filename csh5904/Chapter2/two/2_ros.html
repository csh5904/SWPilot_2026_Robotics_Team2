1. 로봇 운영체제(ROS)의 개념

  • 정의: 로봇 운영체제(Robot Operating System, ROS)는 하드웨어 제어용 OS라기보다는, 리눅스(Linux) 등 기존 OS 위에서 작동하는 로봇 전용 미들웨어(Middleware)이자 소프트웨어 프레임워크입니다.
  • 목적: 하드웨어 추상화, 분산 프로세스 간 메시지 패싱, 패키지 관리 등을 제공하여 로봇 소프트웨어의 재사용성을 극대화합니다.

2. 운영체제 사용 유무에 따른 로봇의 차이

구분 운영체제(ROS 등)를 사용하는 로봇 운영체제(ROS)를 사용하지 않는 로봇
제어 방식 고성능 프로세서(SBC, AP) 위에서 프로세스 단위 분산 제어 마이크로컨트롤러(MCU) 기반의 펌웨어(Firmware) 제어
소프트웨어 구조 독립된 노드들이 실시간 메시지 통신(Pub/Sub)으로 유기적 작동 하나의 거다란 메인 루프(while(1)) 내 순차 제어
확장성 이미 검증된 내비게이션(Nav2) 등의 패키지를 즉시 활용 가능 하드웨어 의존적 코드를 일일이 수정해야 하므로 재사용성 낮음
주요 분야 자율주행 로봇(AMR), 로봇 팔(Manipulator), 드론 등 고차원 연산 청소기, 라인트레이서 등 단일 목적의 임베디드 시스템

3. 우분투 리눅스(Ubuntu 22.04)에 ROS2 Humble 설치 방법

  1. 로케일(Locale) 설정: ko_KR.UTF-8 환경을 빌드합니다.
  2. 저장소 및 APT 키 등록: Ubuntu Universe 저장소를 활성화하고 ROS2 공식 인증 키를 추가합니다.
  3. 소스 리스트 추가: ROS2 패키지 저장소를 시스템 소스 리스트에 등록합니다.
  4. 데스크톱 버전 설치: sudo apt install ros-humble-desktop 명령어로 전체 환경을 설치합니다.
  5. 환경 변수 자동화: 터미널 실행 시 자동 로드를 위해 ~/.bashrc 하단에 source /opt/ros/humble/setup.bash를 추가합니다.

4. ROS2 패키지(Package)와 노드(Node)의 관계

  • 노드 (Node): 센서 데이터 읽기, 모터 제어 등 특정 기능을 수행하는 최소 실행 단위의 프로그램(프로세스)입니다.
  • 패키지 (Package): 하나 이상의 노드와 설정 파일, 빌드 규칙을 모아놓은 소프트웨어 컨테이너(폴더)입니다.
  • 관계: 개발자는 패키지 단위로 코드를 배포·관리하며, 실행 시에는 패키지 내부의 노드들을 각각 구동하여 시스템을 완성합니다.

5. `ros2 run` 명령어와 토커/리스너 데모 실습 결과

  • 명령어 구조: ros2 run [패키지_이름] [노드_이름]
  • 데모 요약: demo_nodes_cpp 패키지의 talker 노드가 메시지를 발행(Publish)하면, 다른 터미널에서 실행된 listener 노드가 이를 수신(Subscribe)하여 화면에 출력합니다.
