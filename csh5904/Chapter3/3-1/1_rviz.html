3-1. RViz2 사용법과 TF(Transform Frames)

Chapter 3 · 문제 1

과제 목표
RViz2를 실행해서 튜토리얼 로봇을 여러 방식으로 살펴보고, TF의 세 가지 핵심 개념(좌표계·변환·트리)이 무엇인지 조사한다. TF 화면에서 좌표축 색이 뜻하는 바도 확인한다.

1. TF의 개요와 3대 개념

TF는 로봇 안에 있는 여러 부품(센서, 링크, 바퀴, 세계 기준점 등)이 서로 어디에 어떤 자세로 놓여 있는지를 실시간으로 계산해 주는 좌표 변환 관리 시스템이다.

① 좌표계 (Coordinate Frames)

로봇의 특정 부위나 센서, 환경의 기준점을 중심으로 잡은 3차원 좌표계(X, Y, Z)다. 로봇 중심인 base_link, 라이다 기준인 laser_frame, 세계 기준인 map·odom 등이 대표적이다.

② 변환 (Transforms)

한 좌표계에서 다른 좌표계로 이동(Translation)과 회전(Rotation)을 통해 넘어가는 수학적 관계다. 예를 들어 laser_frame에서 측정한 장애물 위치를 base_link 기준으로 바꿀 때 쓴다. 시간 정보를 포함하므로 과거 특정 시점의 변환도 계산할 수 있다.

③ 트리 (TF Tree)

시스템 안의 모든 좌표계가 부모-자식 관계로 연결된 하나의 계층 구조다. 자식 좌표계는 부모를 하나만 가질 수 있고, 순환(loop) 구조를 만들면 안 된다. 이 트리 덕분에 어떤 두 좌표계 사이든 끊기지 않고 변환을 추적할 수 있다.

2. 시뮬레이션 환경에서 TF의 역할

Gazebo 같은 시뮬레이션에서 TF는 가상 세계의 물리 상태와 제어 소프트웨어를 이어 주는 다리 역할을 한다.

  • 센서 데이터 정렬: 카메라·라이다가 측정한 값을 센서 위치(camera_link)에서 로봇 전체 기준(base_link)으로 변환해 올바른 위치에 매핑한다.
  • 상태 추정(Odometry): 바퀴 회전수 등을 근거로 odom → base_link 변환을 계속 브로드캐스트해서, 로봇이 어디로 움직이는지 소프트웨어에 알려준다.

3. 실행 방법

튜토리얼 패키지를 설치한 뒤 아래 명령으로 RViz2를 띄운다.

sudo apt install ros-humble-urdf-tutorial
ros2 launch urdf_tutorial display.launch.py \
  model:=/opt/ros/humble/share/urdf_tutorial/urdf/08-macroed.urdf.xacro

Displays > RobotModel > Links에서 항목을 켜고 끄면서 로봇이 어떤 구조로 되어 있는지 확인하고, RobotModel 선택을 끈 뒤 TF를 켜서 base_link만 남겨 좌표축을 확인한다.

4. 좌표축 색(빨강·초록·파랑)의 의미

RViz2를 포함한 대부분의 로봇 소프트웨어는 RGB-XYZ 규칙을 따른다.

축색방향
X축빨강 (Red)앞
Y축초록 (Green)왼쪽
Z축파랑 (Blue)위

암기 팁: RGB 순서 그대로 X, Y, Z에 대응된다.
