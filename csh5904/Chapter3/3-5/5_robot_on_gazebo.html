3-5. Gazebo와 ROS2 연동 (원 주행)

Chapter 3 · 문제 5

과제 목표
URDF의 <inertial>·<collision> 태그와 관련 물리 개념을 조사한다. 로봇 모델에 이 태그를 추가하고, Gazebo와 ROS2가 연동되는 방식·플러그인(특히 diff_drive_controller)을 알아본 뒤, 로봇이 원을 그리며 돌도록 제어 노드를 만든다.

1. collision · inertial 태그와 물리 개념

collision

로봇이 바닥·장애물·다른 링크와 부딪힐 때 계산되는 물리적 외형이다. <visual>에는 복잡한 메쉬를 써도, <collision>에는 Box·Cylinder·Sphere 같은 기본 도형을 쓰는 게 성능에 좋다.

inertial

물리 엔진이 뉴턴-오일러 방정식을 풀 때 필요한 데이터다.

  • <mass> : 링크 질량(kg)
  • <inertia> : 회전 저항을 나타내는 3×3 대칭 관성 텐서 (대칭성으로 6개 성분만 정의). 주축이 질량 중심과 맞으면 교차 성분 ixy, ixz, iyz는 0으로 둔다.
  • <origin> : 기하학적 중심이 아닌 실제 무게 중심(CoM) 위치

주요 형상의 관성 모멘트 공식:

  • 상자(가로 w, 세로 d, 높이 h, 질량 m): ixx = (1/12)m(d²+h²), iyy = (1/12)m(w²+h²), izz = (1/12)m(w²+d²)
  • 원기둥(반지름 r, 높이 h, 질량 m, z축): ixx = iyy = (1/12)m(3r²+h²), izz = (1/2)mr²

태그 추가 예시

<link name="base_link">
  <visual>
    <geometry><box size="0.5 0.3 0.1"/></geometry>
  </visual>
  <collision>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <geometry><box size="0.5 0.3 0.1"/></geometry>
  </collision>
  <inertial>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <mass value="5.0"/>
    <inertia ixx="0.0416" ixy="0.0" ixz="0.0" iyy="0.1083" iyz="0.0" izz="0.1333"/>
  </inertial>
</link>

2. ROS2와 Gazebo 연동 및 로봇 배치(Spawn)

ROS2와 Gazebo는 gazebo_ros_pkgs 메타 패키지로 통신한다. gazebo_ros의 전용 플러그인들이 Gazebo 내부 데이터(센서값, 조인트 상태)를 ROS2 토픽(/odom, /scan, /tf)으로 바꾸고, 반대로 ROS2의 제어 명령(/cmd_vel)을 물리 엔진에 전달한다.

로봇 배치는 보통 이렇게 한다.

  1. robot_state_publisher가 URDF를 읽어 robot_description으로 발행
  2. gazebo_ros의 spawn_entity.py에 그 내용을 넘겨 Gazebo 안에 로봇 생성

3. Gazebo 플러그인과 diff_drive_controller

플러그인은 시뮬레이터 기능(물리 바인딩, 센서 생성, 모터 제어)을 확장하는 C++ 모듈이다. URDF 안에 <gazebo> 태그로 선언하며, 이게 없으면 URDF 모델은 그냥 움직이지 못하는 인형일 뿐이다.

libgazebo_ros_diff_drive.so는 좌우 두 바퀴 로봇을 제어하는 표준 플러그인으로, geometry_msgs/Twist 타입인 /cmd_vel을 구독해 로봇을 움직인다.

<gazebo>
  <plugin name="differential_drive_controller" filename="libgazebo_ros_diff_drive.so">
    <update_rate>30</update_rate>
    <left_joint>left_wheel_joint</left_joint>
    <right_joint>right_wheel_joint</right_joint>
    <wheel_separation>0.35</wheel_separation>
    <wheel_diameter>0.15</wheel_diameter>
    <max_wheel_torque>20</max_wheel_torque>
    <max_wheel_acceleration>1.0</max_wheel_acceleration>
    <command_topic>cmd_vel</command_topic>
    <odometry_topic>odom</odometry_topic>
    <odometry_frame>odom</odometry_frame>
    <robot_base_frame>base_footprint</robot_base_frame>
    <publish_odom>true</publish_odom>
    <publish_odom_tf>true</publish_odom_tf>
    <publish_wheel_tf>true</publish_wheel_tf>
  </plugin>
</gazebo>

4. 원 주행 제어

제어 노드(circle_drive_node.py)에서 /cmd_vel에 직진 속도(linear.x)와 회전 속도(angular.z)를 일정하게 함께 실어 보내면, 로봇이 일정한 반지름의 원을 그리며 돈다. 원의 크기는 두 값의 비율(v / ω)로 조정한다.
