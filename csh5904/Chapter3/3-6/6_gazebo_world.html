3-6. Gazebo 월드(환경) 만들기

Chapter 3 · 문제 6

과제 목표
Gazebo 환경(World) 파일이 뭔지, 어떻게 만드는지 조사한다. 월드 좌표계와 로컬 좌표계의 관계도 알아본다. 평탄한 바닥에 원형 경로를 가진 월드를 만들고, 로봇이 그 경로 위에서 원을 그리며 돌도록 배치한다.

1. 월드(World) 파일이란

Gazebo에서 로봇이 움직이는 가상 공간(지면·조명·장애물 등)을 정의하는 파일이다. XML 기반 SDF(Simulation Description Format) 형식을 쓰고 확장자는 .world다.

  • <physics> : 중력(-9.81 m/s²), 타임 스텝 등
  • <light> : 태양광·포인트 조명 등 광원
  • <include> : 기본 지면(ground_plane)이나 외부 모델 호출
  • <plugin> : 월드 전체 상태를 제어·모니터링하는 스크립트

2. 월드를 만드는 방법

방법특징용도
텍스트 에디터 직접 작성SDF 코드를 직접 작성. 수치 제어가 정밀하지만 시각 배치가 어려움정밀한 물리 제어, 빈 월드
Gazebo GUI 도구Building/Model Editor로 드래그해서 배치. 직관적실내 지도, 미로·방 구조
3D 모델링 툴 연동Blender·CAD로 만든 뒤 .dae/.stl로 내보내 SDF로 감쌈복잡한 지형, 정교한 환경
알고리즘 자동 생성파이썬 등으로 장애물을 규칙·랜덤 배치SLAM·내비 학습용 랜덤 맵

3. 월드 좌표계와 로컬 좌표계

월드 좌표계

가상 공간 전체의 절대 기준이 되는 고정 좌표계다. 변하지 않는 원점 (0,0,0)을 가지며, 로봇의 절대 위치(Odometry Ground Truth)를 잴 때 기준이 된다.

로컬 좌표계

로봇 본체나 특정 링크의 중심을 원점으로 삼는 상대 좌표계다. 로봇이 움직이면 함께 움직인다. ROS 표준으로 X는 앞, Y는 왼쪽, Z는 위다.

두 좌표계의 관계

로봇이 월드 안에서 움직이면, 로컬↔월드 관계는 이동과 회전 행렬의 조합(동차 변환)으로 계산된다. 센서가 로컬 기준으로 잡은 장애물을, 로봇의 현재 자세를 곱해 월드 절대 위치로 바꾼다. 회전은 짐벌락을 피하려고 주로 쿼터니언(x, y, z, w)으로 표현한다.

4. 월드와 함께 시뮬레이터 실행하기

터미널에서 월드 경로를 직접 넘길 수도 있고, ROS2 Launch로 넘길 수도 있다(권장).

# CLI에서 직접
gazebo --verbose /path/to/worlds/custom.world
# ROS2 Launch에서 gzserver에 world 경로 주입
start_gazebo_server = IncludeLaunchDescription(
    PythonLaunchDescriptionSource(
        os.path.join(pkg_gazebo_ros, 'launch', 'gzserver.launch.py')),
    launch_arguments={'world': world_path}.items()
)

5. 원형 경로 월드 + 로봇 배치

평탄한 바닥 위에 이전 문제에서 로봇이 그리던 원과 같은 크기의 원형 경로를 만들었다. 로봇의 초기 위치·방향은 제어 노드에서 지정하고, 시뮬레이터에서 주행 결과를 보면서 값을 맞춰 경로 위를 계속 돌게 했다. 실행이 시작되면 사용자가 멈출 때까지 원 주행을 반복한다.
