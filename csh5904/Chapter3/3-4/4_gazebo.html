3-4. Gazebo 물리 시뮬레이션

Chapter 3 · 문제 4

과제 목표
Gazebo가 뭔지, RViz2와 어떻게 다른지 알아본다. Gazebo를 설치·실행해서 박스와 구를 놓고 중력·충돌·반발 같은 물리 현상을 직접 테스트한다.

1. Gazebo와 RViz2의 차이

Gazebo는 중력·마찰·관성·탄성 같은 실제 물리 법칙이 적용되는 물리 시뮬레이터다. 반면 RViz2는 시뮬레이터가 아니라 로봇 내부 데이터(센서값, TF, 경로 등)를 눈으로 보기 쉽게 그려주는 시각화 도구다.

2. 설치 및 실행

ROS2 Humble에 맞는 Gazebo와 연동 패키지를 설치한다. 명령 끝의 와일드카드(*)를 빼먹지 않아야 한다.

sudo apt update
sudo apt install ros-humble-gazebo* -y
gazebo

3. 기본 물리 테스트

박스 배치와 축 색 비교

상단 툴바에서 Box를 놓고 이동 아이콘으로 클릭하면 축 기즈모가 나온다. Gazebo도 RViz2와 같은 ROS 표준(오른손 법칙)을 따르므로 색이 똑같다 — X는 빨강(앞), Y는 초록(왼쪽), Z는 파랑(위).

자유 낙하

Z축(파랑)을 잡고 박스를 위로 올렸다 놓으면 중력(약 -9.8 m/s²)을 받아 바닥으로 떨어진다. 물리 엔진이 실시간으로 돌고 있다는 뜻이다.

Pause 테스트

하단 Pause를 누른 상태에서 박스를 위로 올리면 허공에 그대로 멈춰 있다(물리 시간 정지). Play를 다시 누르면 그 순간부터 떨어진다.

4. 구(Sphere)를 이용한 충돌 테스트

Pause 상태에서 구를 박스 위 적당한 높이에 놓고 Play를 누르면, 구가 떨어져 박스 윗면과 부딪혀 얹힌다. 구를 박스 모서리 쪽으로 살짝 치우쳐 놓으면, 떨어지며 회전 관성이 생겨 옆으로 데굴데굴 굴러 바닥으로 떨어진다.

5. Insert 탭 물체와 탄성체 확인

왼쪽 Insert 탭에서 여러 물체를 추가할 수 있다. Construction Barrel, Dumpster 같은 기성 모델은 대부분 바닥에 닿으면 튕기지 않고 멈춘다(반발 계수 0).

반면 cricket_ball이나 ball_bearing은 SDF 충돌 속성에 <bounce>와 <restitution_coefficient>(반발 계수) 값이 높게(0.7~0.9) 들어 있어서, 바닥에 닿으면 통통 튀어 오른다. 기본 구형 모델과의 차이가 바로 이 반발 계수다.
