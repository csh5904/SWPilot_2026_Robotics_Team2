# 4_gazebo.md

## 1. Gazebo와 RViz2의 개념 및 차이점

ROS2 에코시스템에서 로봇을 시각화하고 검증하는 데 쓰이는 두 가지 핵심 도구인 **Gazebo**와 **RViz2**는 목적과 작동 방식에서 뚜렷한 차이가 있습니다.

### 1) Gazebo 시뮬레이터란?

Gazebo는 실제 물리 법칙(중력, 마찰력, 관성, 탄성 등)이 적용된 가상의 3차원 환경을 제공하는 "물리 시뮬레이터"입니다. 로봇이 가상의 공간 내에서 부딪히고, 바퀴가 미끄러지고, 센서(카메라, LiDAR)로 가상 환경을 측정하는 등 **실제 로봇을 구동하기 전 하드웨어와 주변 환경을 모사**할 때 사용합니다.

### 2) RViz2 시뮬레이터란?

정확히 표현하면 RViz2는 시뮬레이터가 아니라 "3D 데이터 시각화 도구(Visualizer)"입니다. 로봇이 실제로든 가상으로든 움직이고 있을 때, 센서가 바라보는 거리 데이터(Point Cloud), 현재 계산된 로봇의 좌표 트리(`/tf`), 이동 경로(Path) 등 **로봇 내부의 소프트웨어 데이터를 개발자가 눈으로 보기 쉽게 그래픽으로 뿌려주는 역할**을 합니다.

### 📊 Gazebo vs RViz2 비교 요약

| 항목 | Gazebo | RViz2 |
| --- | --- | --- |
| **주요 목적** | 물리 법칙이 적용된 가상 세계 및 로봇 구동 환경 시뮬레이션 | 로봇 내부의 센서 데이터 및 상태 정보 시각화 |
| **물리 엔진** | 있음 (중력, 충돌, 반발력, 마찰 등 연산) | 없음 (데이터가 지시하는 위치에 그대로 렌더링) |
| **주요 입력** | 제어 명령 토픽 (`/cmd_vel` 등) | 센서 데이터 토픽 (`/scan`, `/tf`, `/odom` 등) |
| **가상 센서** | 가상 환경의 장애물을 감지하여 센서 값 생성 | 생성된 센서 값을 화면에 점이나 선으로 표시 |

## 2. Gazebo 설치 및 실행

### 1) Gazebo 설치 명령어

ROS2 Humble 버전에 대응하는 Gazebo 및 ROS 연동 패키지를 설치합니다. 명령어 끝의 와일드카드(`*`)를 포함하여 관련 라이브러리를 전부 설치해야 합니다.

```bash
sudo apt update
sudo apt install ros-humble-gazebo* -y

```

### 2) Gazebo 실행

설치가 완료되면 터미널에 아래 명령어를 입력하여 시뮬레이터를 실행합니다. (최초 실행 시 가제보 공식 모델 라이브러리를 불러오느라 수십 초 이상 지연될 수 있습니다.)

```bash
gazebo

```

## 3. 기본 물리 특성 실습 및 축 색상 비교

### 1) 박스 배치 및 축 색상 비교

1. Gazebo 상단 툴바의 **Box(정육면체)** 아이콘을 선택하여 격자판 아무 곳에나 배치합니다.
2. 상단 메뉴의 이동 아이콘(화살표 십자가 모양)을 클릭한 후, 배치한 박스를 클릭하면 3차원 축 기즈모가 나타납니다.

#### 🎨 RViz2 TF 축과 Gazebo 축의 색상 비교

* 두 시뮬레이터 모두 ROS 표준 3D 좌표계 규격(오른손 법칙)을 따르므로 색상이 완벽히 일치합니다.
* **X축:** **빨간색 (Red)** $\rightarrow$ 전방 방향
* **Y축:** **초록색 (Green)** $\rightarrow$ 좌측 방향
* **Z축:** **파란색 (Blue)** $\rightarrow$ 상단 방향

> 💡 **암기 팁:** RGB 순서대로 X(R), Y(G), Z(B) 매칭됩니다.

### 2) 박스의 자유 낙하 실험

* Z축(파란색 화살표)을 마우스로 클릭하고 위로 끌어올렸다가 마우스를 놓으면, **박스가 가제보 월드의 중력(약 $-9.8m/s^2$)을 받아 바닥으로 뚝 떨어지는 현상**이 발생합니다.
* 이는 물리 엔진이 실시간으로 구동되고 있음을 보여줍니다.

### 3) 시뮬레이션 일시 정지(Pause) 실험

1. Gazebo 하단 툴바 좌측의 **Pause(일시정지, `||` 모양)** 아이콘을 클릭합니다.
2. 이 상태에서 박스를 Z축 위로 높게 이동시키면, 마우스를 놓아도 박스가 허공에 그대로 멈춰 있습니다. (물리 시간 연산이 중단되었기 때문)
3. 하단의 버튼이 Play(재생, `▶` 모양)로 바뀐 것을 확인하고 클릭하면, 멈춰있던 박스가 그 순간부터 중력을 받아 다시 바닥으로 낙하합니다.

## 4. 구(Sphere) 모델을 이용한 충돌 시뮬레이션

1. 먼저 하단의 **Pause** 아이콘을 눌러 시뮬레이션을 잠시 멈춥니다.
2. 상단 메뉴에서 **Sphere(구)** 아이콘을 선택하여 배치합니다.
3. 이동 툴을 이용해 구의 위치를 **아래에 있는 박스 바로 위의 적당한 공중 높이**로 이동시킵니다.
4. 하단의 **Play** 아이콘을 클릭합니다.

### 🧐 시뮬레이션 결과 및 상황 확인

* **충돌 및 안착:** 구가 중력으로 떨어지다가 아래에 있는 박스의 상단 면과 부딪힙니다. 박스와 구의 충돌 외곽선(Collision)이 맞물리면서 구는 박스 위에 툭 하고 얹혀 멈추게 됩니다.
* **굴러 떨어지기 연출:** 다시 시뮬레이션을 멈추거나 이동 툴로 구의 위치를 **박스의 모서리 경계면 쪽**으로 살짝 치우치게 옮긴 후 실행하면, 구가 떨어지며 박스 모서리에 부딪힌 뒤, 구형 기하학(Geometry) 구조에 따라 회전 관성이 발생하여 **박스 옆으로 데굴데굴 굴러 바닥으로 떨어지는 물리 현상**이 정확히 시뮬레이션됩니다.

## 5. Insert 탭을 통한 다양한 물체 추가 및 탄성체 확인

Gazebo 왼쪽 패널의 **Insert** 탭을 확장하면, 기본 가제보 데이터베이스나 내 컴퓨터에 저장된 다양한 3D 모델 요소(장애물, 가구, 차량 등)를 월드에 추가할 수 있습니다.

### 1) 다양한 물체 배치 실험

* `Construction Barrel`(공사용 드럼통), `Dumpster`(쓰레기통) 등 여러 물체를 공중에 띄웠다 떨어뜨려 보면, 대부분은 바닥에 닿는 순간 쿵 소리를 내듯 단단하게 박히며 전혀 튕기지 않고 멈춥니다.
* 이는 일반적인 기성 모델들의 물리 속성 중 반발 계수(탄성)가 `0`으로 세팅되어 있기 때문입니다.

### 2) 기본 구형 모델과 달리 통통 튀는 탄성 물체 확인

* **해당 물체 이름:** **`cricket_ball`** (크리켓 공) 또는 **`ball_bearing`** (볼베어링/쇠구슬)
* **물리적 특성의 차이점 (`bounce` 속성):**
일반적인 구형(Sphere) 모델은 충돌 시 에너지를 흡수하여 튕기지 않지만, `cricket_ball`이나 `ball_bearing` 같은 모델은 SDF 설정 파일 내부의 충돌 속성에 **`<bounce>`** 태그와 **`<restitution_coefficient>`(반발 계수)** 값이 높게(예: `0.7 ~ 0.9`) 주입되어 있습니다.
* 이 반발 계수로 인해 충돌 시 운동에너지가 보존되어 바닥에 닿았을 때 통통 튕겨 올라오는 탄성 충돌 시뮬레이션이 연출됩니다.
