3-9. PID 파라미터 튜닝

Chapter 3 · 문제 9

과제 목표
PID 튜닝 과정과 P·I·D 간의 상호작용에서 주의할 점을 조사한다. P·I·D를 ROS2 파라미터로 선언해 실행 중에 실시간으로 값을 바꾸며 튜닝하고, 튜닝 전/후의 주행 성능을 원형 경로와 복잡한 경로에서 비교한다.

1. P·I·D 간의 상호작용과 주의점

세 값은 서로 영향을 준다. 하나를 키우면 다른 항의 효과도 달라지므로, 한 번에 하나씩 조금씩 바꾸는 게 핵심이다.

파라미터올리면너무 크면
Kp반응이 빨라짐, 정상상태 오차 감소오버슛·진동 발생
Ki남아있는 오차 제거적분 누적으로 오버슛·불안정(적분 와인드업)
Kd진동·오버슛 억제, 안정화센서 잡음에 민감, 반응 둔화

특히 P를 너무 올린 상태에서 I까지 키우면 진동이 급격히 심해진다. D는 잡음을 증폭할 수 있으니 마지막에 조금씩 올린다.

2. 대표적인 튜닝 방법

  • 시행착오(Trial and Error): P부터 올려 반응을 보고, I·D를 차례로 조정한다. 가장 직관적이고 실무에서 많이 쓴다.
  • 지글러-니콜스(Ziegler–Nichols): I·D를 0으로 두고 P만 올려 시스템이 일정하게 진동하기 시작하는 임계 게인(Ku)과 진동 주기(Tu)를 찾은 뒤, 공식으로 Kp·Ki·Kd를 계산한다.
  • 소프트웨어 자동 튜닝: 성능 지표(오차 적분 등)를 최소화하도록 알고리즘이 값을 탐색한다.

3. ROS2 파라미터로 실시간 튜닝

P·I·D를 코드에 박아두면 값을 바꿀 때마다 다시 빌드해야 한다. ROS2 파라미터로 선언하면 실행 중에 ros2 param 명령으로 바로 바꿀 수 있어 튜닝이 훨씬 빠르다.

파라미터 선언 (노드 안)

self.declare_parameter('kp', 1.0)
self.declare_parameter('ki', 0.1)
self.declare_parameter('kd', 0.6)

self.kp = self.get_parameter('kp').value
self.ki = self.get_parameter('ki').value
self.kd = self.get_parameter('kd').value

실행 중 값 확인·변경

# 현재 값 확인
ros2 param get /line_tracking_pid_node kp

# 실시간으로 값 변경
ros2 param set /line_tracking_pid_node kp 1.4
ros2 param set /line_tracking_pid_node kd 0.8

노드에서 add_on_set_parameters_callback으로 변경을 받아 내부 PID 계수에 즉시 반영하면, 주행 중에도 값이 바뀐다.

4. 튜닝 순서

파라미터는 한 번에 하나씩, 아래 순서로 조정하는 게 좋다.

  1. Ki=0, Kd=0으로 두고 Kp부터. 라인을 따라가되 약간 진동이 남을 정도까지 올린다.
  2. Kd를 조금씩 올린다. P 때문에 생긴 진동·오버슛을 눌러 안정화한다.
  3. 마지막에 Ki를 조금씩 올린다. 커브에서 계속 안쪽/바깥쪽으로 남는 정상상태 오차를 없앤다. 진동이 생기면 다시 줄인다.

5. 튜닝 전/후 성능 비교

원형 경로

튜닝 전에는 커브에서 좌우로 크게 출렁이며 가끔 라인을 놓쳤다. Kp를 적정선까지 올리고 Kd로 진동을 잡은 뒤에는 흔들림이 눈에 띄게 줄고, 라인 중앙을 안정적으로 유지하며 돌았다.

복잡한 경로

직선과 곡률이 다른 커브가 섞인 복잡한 경로(complex_path_world)에서 비교했다. 튜닝 전 파라미터는 곡률이 급한 구간에서 반응이 못 따라가 이탈이 잦았지만, 튜닝 후에는 급커브에서도 회전량이 오차에 맞게 조절돼 경로를 훨씬 오래 유지했다.

테스트에 쓴 복잡 경로 월드는 직선 박스와 반지름이 다른 원기둥(커브)을 SDF로 이어 붙인 폐곡선 형태다.
