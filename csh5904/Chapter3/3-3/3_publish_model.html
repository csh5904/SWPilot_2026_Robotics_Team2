3-3. 로봇 상태 퍼블리셔 구현

Chapter 3 · 문제 3

과제 목표
로봇 상태를 게시한다는 게 무슨 뜻인지, URDF 로봇 상태를 어떻게 게시하는지 조사한다. /joint_states에 관절 상태를 발행하는 노드를 파이썬으로 만들고, robot_state_publisher와 함께 실행해 RViz2에서 로봇을 확인한다.

1. 로봇 상태를 "게시"한다는 것

로봇의 링크들이 지금 3차원 공간에서 서로 어떤 위치와 자세(TF)로 놓여 있는지를 시스템 전체에 실시간으로 공유하는 것을 말한다.

  • 구조 변화 전파: "바퀴가 회전했다", "팔 관절이 45도 꺾였다" 같은 동적 변화를 계속 업데이트한다.
  • 좌표 변환: 상태를 게시해야 센서 측정값을 base_link나 map·odom 기준으로 올바르게 변환할 수 있다.

2. URDF 로봇 상태 게시 흐름

URDF 자체는 멈춰 있는 문서라서, 움직이는 상태를 게시하려면 다음 과정이 필요하다.

  1. Launch 파일로 URDF(또는 Xacro)를 읽어온다.
  2. 읽어온 robot_description을 파라미터로 robot_state_publisher 노드를 실행한다.
  3. 모터 엔코더나 시뮬레이터에서 각 관절의 현재 값을 받아온다.
  4. robot_state_publisher가 구조 + 관절 값을 조합해 /tf를 계산·게시한다.

3. robot_state_publisher와 /joint_states의 관계

구성 요소역할입력출력
/joint_states관절의 동적 상태값 전달엔코더, 시뮬레이터(데이터 메시지)
robot_state_publisher순기구학 계산 → 좌표 트리 생성URDF + /joint_states/tf, /tf_static

정리하면 /joint_states는 "각 관절이 몇 rad/몇 m 움직였는가"라는 숫자만 담고, robot_state_publisher는 URDF 구조와 그 숫자를 조합해 순기구학을 풀어 /tf를 뿜어낸다.

4. joint_state_publisher.py

/joint_states 토픽에 좌우 바퀴 관절 상태를 10Hz로 발행하는 노드다. TF 매칭을 위해 헤더 타임스탬프를 꼭 넣고, URDF에 적힌 관절 이름과 정확히 맞춘다.

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import JointState
import math

class CustomJointStatePublisher(Node):
    def __init__(self):
        super().__init__('custom_joint_state_publisher')
        
        # 1. /joint_states 토픽을 발행할 Publisher 생성
        self.joint_pub = self.create_publisher(JointState, 'joint_states', 10)
        
        # 2. 0.1초(10Hz)마다 토픽을 발행할 타이머 생성
        self.timer = self.create_timer(0.1, self.timer_callback)
        
        # 관절 제어를 위한 임시 변수 (가상으로 움직임을 주기 위함)
        self.pose = 0.0

    def timer_callback(self):
        # 3. JointState 메시지 객체 생성
        msg = JointState()
        
        # 4. 현재 시간 타임스탬프 설정 (TF 매칭을 위해 필수!)
        msg.header.stamp = self.get_clock().now().to_msg()
        
        # 5. URDF에 적힌 정확한 관절 이름 매칭
        msg.name = ['right_wheel_joint', 'left_wheel_joint']
    
        msg.position = [self.pose, self.pose]
        # 7. 메시지 발행
        self.joint_pub.publish(msg)
        self.get_logger().info(f'Publishing Joint States: {msg.position}')
        


def main(args=None):
    rclpy.init(args=args)
    node = CustomJointStatePublisher()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

if __name__ == '__main__':
    main()

5. Launch 파일 (state_publish.launch.py)

Xacro를 파싱해 robot_state_publisher에 주입하고, 직접 만든 joint_state_publisher 노드와 RViz2를 함께 띄운다.

import os
from ament_index_python.packages import get_package_share_directory
from launch import LaunchDescription
from launch_ros.actions import Node
import xacro

def generate_launch_description():
    pkg_share = get_package_share_directory('my_robot_sim')
    xacro_file = os.path.join(pkg_share, 'urdf', 'my_first_robot.urdf.xacro')

    # Xacro 파싱 및 XML 변환
    robot_description_config = xacro.process_file(xacro_file)
    robot_desc = robot_description_config.toxml()

    return LaunchDescription([
        # 1. URDF 설계도를 주입받아 기구학 변환(/tf)을 뿜어내는 노드
        Node(
            package='robot_state_publisher',
            executable='robot_state_publisher',
            name='robot_state_publisher',
            output='screen',
            parameters=[{'robot_description': robot_desc}]
        ),
        
        # 2. 질문자님이 작성한 /joint_states 발행 노드
        Node(
            package='my_robot_sim',
            executable='joint_state_publisher',
            name='custom_joint_state_publisher',
            output='screen'
        ),
        
        # 3. [완벽 수정] 괄호나 콤마 없이 공백과 등호(=)로만 파싱하는 RViz2 인라인 주입 문법
        Node(
            package='rviz2',
            executable='rviz2',
            name='rviz2',
            output='screen'
        )
    ])
