주제 1: 마이크로컨트롤러의 기본 개념과 제어 방식

1. 마이크로컨트롤러의 역할

  • 개념: 마이크로컨트롤러(MCU)는 하나의 칩 안에 프로세서(CPU), 메모리(RAM, ROM), 입력 및 출력 포트가 모두 집약된 '초소형 컴퓨터'이다.
  • 역할: 센서로부터 주변 환경 데이터를 입력받아 미리 작성된 프로그램 코드로 연산을 수행한 뒤, 모터나 발광 다이오드(LED) 같은 구동 장치를 제어하는 로봇의 '두뇌' 역할을 담당한다.

2. 기본 제어 방식과 인터페이스

디지털 핀과 아날로그 핀

  • 디지털 핀: 0V(Low)와 5V 또는 3.3V(High)라는 두 가지 상태만 다룬다. LED를 켜고 끄거나, 스위치가 눌렸는지 판별할 때 사용한다.
  • 아날로그 핀: 0V부터 5V까지 연속적으로 변하는 전압을 읽어 들인다. 내부의 아날로그-디지털 변환기(ADC)를 통해 연속적인 전압 값을 0~1023 등 이산적인 디지털 숫자로 바꾸어 컴퓨터가 인식하게 한다. 거리를 재는 센서나 빛의 세기를 측정하는 센서 등에 쓰인다.

PWM (펄스 폭 변조)

  • 개념: 디지털 핀은 0과 1만 보낼 수 있지만, 신호가 켜지는 시간(High)과 꺼지는 시간(Low)의 비율을 아주 빠르게 조절하여 마치 아날로그 전압이 변하는 것과 같은 효과를 내는 제어 방식이다.
  • 용도: 모터의 회전 속도를 부드럽게 조절하거나 LED의 밝기를 은은하게 제어할 때 필수적으로 사용된다.

주제 2: 마이크로컨트롤러의 통신 방식

마이크로컨트롤러가 주변 센서나 다른 칩들과 데이터를 주고받기 위해 사용하는 대표적인 직렬 통신 방식은 다음과 같다.

1. UART (범용 비동기 송수신)

  • 특징: 데이터를 보내는 선(TX)과 받는 선(RX) 두 개를 교차로 연결하여 통신한다. 데이터를 맞추기 위한 동기화 시계 신호(클럭)선이 없는 '비동기식' 통신이다.
  • 장단점: 선이 적어 연결이 간단하지만, 통신하는 양쪽 장치가 데이터를 주고받는 속도(보레이트)를 미리 똑같이 맞춰야만 데이터가 깨지지 않는다. 주로 1대1 통신에 쓰인다.

2. I2C (집적 회로간 통신)

  • 특징: 시계 신호를 전달하는 클럭 선(SCL)과 실제 데이터가 오고 가는 데이터 선(SDA) 두 개를 사용한다. 하나의 마이크로컨트롤러(마스터)에 여러 개의 센서(슬레이브)를 병렬로 주렁주렁 매달아 통신할 수 있는 '동기식' 방식이다.
  • 장단점: 각 센서마다 고유의 주소(ID)가 있어 2개의 선만으로 수많은 장치를 제어할 수 있으나, 선을 공유하므로 통신 속도가 상대적으로 느리다.

3. SPI (직렬 주변기기 인터페이스)

  • 특징: 클럭 선(SCK), 마스터가 데이터를 보낼 때 쓰는 선(MOSI), 마스터가 데이터를 받을 때 쓰는 선(MISO), 그리고 제어할 장치를 선택하는 칩 선택 선(CS 또는 SS)까지 총 4개의 선을 사용한다.
  • 장단점: 장치가 늘어날 때마다 칩 선택 선이 하나씩 추가되어 선이 많아지지만, 데이터를 보내고 받는 선이 완벽히 분리되어 있어 통신 속도가 세 방식 중 가장 압도적으로 빠르다.

주제 3: 실시간 제어와 타이머

1. 실시간 제어에서 타이머의 개념

  • 로봇을 제어할 때 '정확히 0.01초마다 엔코더 값을 읽어라' 혹은 '정확히 0.05초마다 모터 속도를 수정하라'처럼 시간에 맞추어 정밀하게 동작을 수행해야 할 때가 많다.
  • 프로그램 코드 안에 일반적인 지연 함수(Delay)를 넣으면 그 시간 동안 컴퓨터가 아무 일도 못 하고 멈춰 버린다. 반면 내부에 하드웨어로 존재하는 타이머(Timer)를 쓰면, 컴퓨터 본체는 자기 일을 하다가 정해진 시간이 되는 순간 하던 일을 멈추고 지정된 제어 동작을 칼같이 실행(인터럽트)할 수 있다.

2. 타이머의 일반적인 사용 절차

  1. 타이머 선택: 마이크로컨트롤러 내부에 있는 여러 타이머(예: 타이머 1, 타이머 2 등) 중 사용할 타이머를 지정한다.
  2. 분주비(Prescaler) 설정: 마이크로컨트롤러의 기본 시계 메인 클럭(예: 16MHz)은 너무 빨라서 시간을 세기 어렵다. 이를 8, 64, 256 등으로 나누어 타이머가 시계추를 똑딱이는 속도를 적절히 늦춰준다.
  3. 목표 카운트 값 설정: 타이머의 내부 숫자가 어디까지 올라갔을 때 신호를 줄 것인지 임계값을 등록한다. 이 값에 따라 주기(예: 1ms 또는 10ms)가 결정된다.
  4. 인터럽트 서비스 루틴(ISR) 작성: 정해진 시간에 도달해 타이머 알람이 울렸을 때, 로봇이 즉각적으로 실행해야 하는 핵심 제어 코드(센서 읽기, 출력 계산 등)를 전용 함수 안에 작성한다.
  5. 타이머 가동: 타이머를 활성화하여 실시간 주기 제어를 시작한다.

주제 4: 아두이노

1. 마이크로보드와 아두이노의 개념

  • 마이크로보드: 알맹이 칩인 마이크로컨트롤러(MCU)를 사용자가 전선 연결과 전원 공급을 쉽게 할 수 있도록 전원 회로, 회로 기판, 단자대 등을 주변에 배치해 놓은 하드웨어 보드 전체를 뜻한다.
  • 아두이노: 이 마이크로보드를 포함하여, 전용 소프트웨어(IDE)와 사용하기 쉬운 프로그래밍 언어 환경까지 통틀어 일컫는 오픈소스 전자 제어 플랫폼이다. 복잡한 레지스터 설정을 몰라도 함수 몇 줄만으로 로봇 하드웨어를 쉽게 제어할 수 있어 전 세계적으로 널리 쓰인다.

2. 아두이노의 인터페이스 및 통신 방식

  • 아두이노 보드(예: 아두이노 우노 기준)는 외곽 테두리에 헤더 핀 형태로 다양한 인터페이스를 제공한다.
  • 입출력 핀: 디지털 데이터를 다루는 0번~13번 핀이 있으며, 이 중 물결표시(~)가 있는 핀들은 앞서 설명한 PWM(펄스 폭 변조) 출력이 가능하다. 아래쪽에는 아날로그 센서 값을 읽을 수 있는 A0~A5 핀이 배치되어 있다.
  • 통신 기능: 아두이노는 컴퓨터와 프로그램 코드를 주고받거나 모니터링을 하기 위해 기본적으로 내장된 UART 통신(0번 RX, 1번 TX 핀 및 USB 포트)을 사용한다. 또한 하드웨어 핀 배열을 통해 센서 인터페이스용인 I2C(A4 SDA, A5 SCL 핀)와 고속 통신용인 SPI(11번 MOSI, 12번 MISO, 13번 SCK 핀 등) 방식을 모두 기본 인터페이스로 지원하므로 다양한 모듈을 손쉽게 붙여 제어할 수 있다.
